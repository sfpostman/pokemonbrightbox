# Setup & Configuration

## Prerequisites

This CLI requires `Go >= 1.19.0` to be installed on your system.

You can download and install Go from the [official Go installation page](https://go.dev/doc/install).

## Installation

To build and install the CLI, run the following commands from the root of the project:

```bash
go mod tidy
go build .
```

This will produce an executable binary in the current directory. You can then run it directly:

```bash
./e2e-library --help
```

To make the CLI available globally, move the binary to a directory in your `PATH`:

```bash
mv ./e2e-library /usr/local/bin/e2e-library
```

## Authentication

The e2e-library CLI stores your credentials locally in `~/.e2e-library/credentials`. You can set up authentication by running the `setup-auth` command.

```bash
e2e-library setup-auth --help
```

Credentials are resolved in the following order (highest priority first):

1. Environment variables (prefix: `E2E_LIBRARY_`)
2. Config file (`./config/config.yaml`)
3. Saved credentials (`~/.e2e-library/credentials`, managed by `setup-auth`)
4. Built-in defaults

### API Key Authentication

To configure API Key Authentication:

```bash
e2e-library setup-auth api-key [value]
```

Or via environment variable:

| Variable              | Description                |
| --------------------- | -------------------------- |
| `E2E_LIBRARY_API_KEY` | API key for authentication |

## Agent Skills

This CLI includes pre-built **skills** — structured, machine-readable documents that describe how to invoke each command. Skills are designed for use with AI agents and coding assistants (e.g., Claude, Cursor, Copilot) to enable accurate, context-aware command generation without hallucination.

Each skill describes a specific CLI command in detail: its purpose, required and optional parameters, expected inputs, and usage examples. When an AI agent loads a skill before invoking a command, it gains the full context needed to construct correct invocations on the first attempt.

Skills are organized by command group at `.claude/skills/<command-group>/SKILL.md`, with individual subcommand skills at `.claude/skills/<command-group>/<subcommand>/SKILL.md`.
